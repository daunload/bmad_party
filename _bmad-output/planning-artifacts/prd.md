# Product Requirements Document - party

**Author:** Seodaun
**Date:** 2025-12-27

## Executive Summary

Party는 파티를 전문적으로 다루는 커뮤니티 플랫폼입니다. 당근마켓과 같은 범용 플랫폼과 달리, 파티에 특화된 기능과 인기도 기반 평가 시스템을 통해 신뢰할 수 있는 참여자들을 확보하고 클린한 파티 커뮤니티를 구축합니다. 파티 생성, 검색, 관리 도구와 파티 후 평가/리뷰 기능을 제공하여 매너 있는 참여자들이 안전한 파티 환경에서 만날 수 있도록 합니다.

현재 사람들은 파티를 찾기 위해 당근마켓과 같은 범용 플랫폼을 사용하고 있지만, 파티에 특화된 기능이 부족하고 파티 참여자들의 신뢰도와 매너를 관리할 수 있는 체계가 없습니다. 이로 인해 파티 주최자와 참여자 모두가 신뢰할 수 있는 참여자를 찾기 어렵고, 안전하고 클린한 파티 환경을 보장하기 어려운 상황입니다.

Party는 이러한 문제를 해결하기 위해 파티만을 전문적으로 다루는 커뮤니티 플랫폼으로 설계되었습니다. 핵심 기능으로는 파티 특화 기능(생성, 검색, 관리), 인기도 기반 평가 시스템, 참여 이력 관리, 파티 후 평가/리뷰, 신규 사용자 온보딩, 공정한 평가 시스템, 파티 주최자 지원 도구가 포함됩니다.

### What Makes This Special

Party의 핵심 차별화 요소는 다음과 같습니다:

1. **파티 전문성**: 파티에만 집중하여 특화된 기능과 경험 제공
2. **인기도 기반 참여 제한**: 낮은 인기도를 가진 참여자의 파티 참여 제한을 통한 품질 관리
3. **신뢰 시스템**: 참여 이력 관리와 평가 시스템을 통한 신뢰할 수 있는 참여자 확보
4. **클린 커뮤니티**: 매너 있는 참여자들과 안전한 파티 환경을 보장하는 커뮤니티 문화
5. **공정하고 투명한 평가**: 평가 기준 명확화, 악의적 평가 탐지, 인기도 회복 메커니즘을 통한 공정성 보장
6. **신규 사용자 친화적**: 신규 사용자 전용 파티와 멘토링 시스템을 통한 진입 장벽 완화

## Product Scope

### MVP - Minimum Viable Product

MVP는 핵심 문제를 해결하고 사용자에게 실질적인 가치를 제공하는 최소 기능 세트에 집중합니다. 파티 생성/검색/관리 도구, 인기도 기반 평가 시스템, 파티 후 평가/리뷰 기능이 MVP의 핵심입니다.

#### Core Features

**1. 파티 생성/검색/관리 도구**
- 파티 생성: 파티 주최자가 파티 정보 입력 및 생성 (제목, 설명, 날짜, 시간, 장소, 인원, 카테고리, 이미지)
- 파티 검색: 참여자가 원하는 파티 검색 (카테고리, 날짜, 지역, 키워드 등)
- 파티 관리: 참여자 관리 (참여 승인/거절), 파티 정보 수정 및 삭제, 파티 상태 관리

**2. 인기도 기반 평가 시스템**
- 인기도 계산: 참여자들의 평가로 인기도 결정 (평균 평가 점수 기반)
- 인기도 표시: 사용자 프로필, 파티 참여자 목록에 인기도 표시
- 인기도 기반 필터링: 파티 주최자가 인기도 기준 설정 가능, 인기도가 낮은 참여자의 파티 참여 제한

**3. 파티 후 평가/리뷰**
- 평가 작성: 파티 후 참여자 간 상호 평가 작성 (매너, 참여도, 분위기 기여 등)
- 리뷰 기능: 파티 및 파티 주최자에 대한 리뷰 작성 및 조회
- 평가 관리: 평가 이력 조회, 받은 평가 및 작성한 평가 목록, 평가 통계 및 요약

#### MVP Success Criteria

MVP의 성공 여부는 다음 기준으로 평가됩니다:

1. **사용자 채택**: MVP 출시 후 3개월 내 월간 활성 사용자 500명 이상 달성, 첫 파티 생성/참여 사용자 비율 30% 이상
2. **핵심 기능 사용**: 파티 생성 기능 70% 이상 사용, 파티 검색 기능 80% 이상 사용, 평가 시스템 60% 이상 사용
3. **초기 유지율**: 7일 유지율 40% 이상, 30일 유지율 25% 이상
4. **사용자 만족도**: 파티 후 긍정적 평가 비율 70% 이상, 사용자 만족도 설문 4.0/5.0 이상
5. **기술적 실현 가능성**: 핵심 기능이 안정적으로 작동, 주요 버그 및 성능 이슈 해결

### Growth Features (Post-MVP)

MVP 성공 후 다음 기능들이 추가될 예정입니다:

**1. 신규 사용자 온보딩**
- 신규 사용자 전용 파티 자동 생성 및 추천
- 경험 있는 사용자 멘토링 시스템
- 신규 사용자 가이드 및 튜토리얼

**2. 공정한 평가 시스템 고도화**
- 악의적 평가 탐지 알고리즘 및 자동 제재
- 인기도 회복 메커니즘 (시간 경과, 추가 긍정 평가)
- 평가 기준 가이드라인 및 예시 제공

**3. 파티 주최자 지원 도구**
- 참여자 모집 알림 및 자동화 기능
- 평가 관리 대시보드
- 인기도 기준 설정 권한 고도화
- 파티 성과 분석 도구

**4. 참여 이력 관리 시스템**
- 참여자의 과거 파티 참여 이력 상세 관리
- 참여 이력 기반 추천 시스템
- 참여 패턴 분석 및 인사이트 제공

## User Journeys

### Journey 1: 파티 주최자 (민수) - 신뢰할 수 있는 참여자 모집하기

민수는 28세 IT 회사 마케터로, 월 2-3회 보드게임 파티를 주최하며 다양한 사람들과 네트워킹을 중시합니다. 그는 매너 있는 참여자들과 즐거운 시간을 보내고 싶어하지만, 당근마켓에서 파티 게시물이 다른 게시물에 묻혀 참여자 모집이 어렵고, 참여자 신뢰도 파악이 어려워 매너 없는 참여자에 대한 걱정이 있습니다.

지인 추천으로 Party를 발견한 민수는 당근마켓의 한계를 느끼던 중 전문 플랫폼을 찾게 되었습니다. 앱 가입 후 프로필을 작성하고 파티 생성 가이드를 확인한 그는 첫 파티를 생성하면서 인기도 기준 설정 방법을 학습합니다.

파티를 생성하고 상세 정보를 입력한 민수는 인기도 기준을 설정하여 참여자를 필터링합니다. 참여자 모집 알림을 받고 관리하면서, 인기도 시스템을 통해 신뢰할 수 있는 참여자들이 자동으로 모집되는 것을 경험합니다. 파티 진행 후, 그는 파티 후 평가 요청을 하고 관리 대시보드를 확인하여 긍정적 평가를 받고 다음 파티 개선점을 파악합니다.

6개월 후, 민수는 정기적으로 파티를 주최하며 커뮤니티의 일부가 되었고, 인기도가 높은 참여자들과 지속적인 관계를 형성하며 평가 관리 대시보드를 통해 파티 품질을 지속적으로 개선하고 있습니다.

**이 여정이 드러내는 요구사항:**
- 사용자 가입 및 프로필 관리
- 파티 생성 및 관리 기능
- 인기도 기준 설정 및 필터링 시스템
- 참여자 모집 알림 시스템
- 파티 후 평가 요청 및 관리 대시보드
- 평가 통계 및 분석 기능

### Journey 2: 파티 참여자 (지은) - 원하는 파티 찾기

지은은 25세 디자이너로, 월 4-5회 다양한 파티에 참여하며 새로운 취미와 사람들을 만나는 것을 좋아합니다. 그녀는 인기도가 높아 많은 파티에 참여할 수 있지만, 당근마켓에서 파티 정보를 찾기 어렵고, 파티 분위기나 참여자 수준을 미리 알기 어려워합니다.

SNS 광고로 Party를 발견한 지은은 당근마켓에서 파티를 찾기 어려워 전문 플랫폼을 찾게 되었습니다. 앱 가입 후 프로필을 작성하고 인기도 시스템 설명을 확인한 그녀는 파티 검색 기능을 학습합니다.

원하는 파티를 검색(카테고리, 날짜, 지역 등)하면서, 지은은 파티 상세 정보와 참여자 인기도를 확인합니다. 파티에 특화된 검색 기능으로 원하는 파티를 쉽게 찾는 순간, 그녀는 인기도 시스템을 통해 신뢰할 수 있는 파티와 참여자들과 만나게 됩니다. 파티 참여 신청 후 참여하고, 파티 후 평가를 작성합니다.

1년 후, 지은은 정기적으로 파티에 참여하며 다양한 사람들과 네트워킹하고, 인기도 유지를 위해 긍정적 평가를 받기 위해 노력하며, 좋아하는 파티 주최자와 지속적인 관계를 형성하고 있습니다.

**이 여정이 드러내는 요구사항:**
- 사용자 가입 및 프로필 관리
- 파티 검색 및 필터링 기능
- 파티 상세 정보 조회
- 인기도 정보 표시
- 파티 참여 신청 기능
- 파티 후 평가 작성 기능

### Journey 3: 신규 사용자 (준호) - 첫 파티 참여하기

준호는 23세 대학생으로, 처음 파티 커뮤니티에 참여하며 새로운 취미와 사람들과의 만남을 원합니다. 하지만 인기도가 없어 일반 파티 참여가 제한될 수 있어 기존 플랫폼의 진입 장벽 때문에 망설이고 있었습니다.

지인 추천으로 Party를 발견한 준호는 앱 가입 후 프로필을 작성하고 신규 사용자 가이드를 확인합니다(인기도 시스템, 평가 방법 등). 신규 사용자 전용 파티를 추천받고 멘토링 시스템 안내를 받습니다.

신규 사용자 전용 파티를 검색하고 참여 신청한 준호는 첫 파티에 참여하면서 멘토의 도움을 받습니다. 파티 후 평가를 받고, 긍정적 평가를 받아 인기도가 상승하면서 일반 파티 참여로 전환할 수 있게 됩니다.

신규 사용자 전용 파티에서 부담 없이 첫 파티를 즐기는 순간, 그리고 첫 파티 후 긍정적 평가를 받고 인기도가 상승하는 순간이 준호에게는 큰 성취감을 줍니다. 일반 파티에 참여할 수 있게 되는 순간, 그는 커뮤니티의 일원이 되었음을 느낍니다.

1년 후, 준호는 정기적으로 파티에 참여하며 커뮤니티의 일부가 되었고, 경험이 쌓이면서 다른 신규 사용자의 멘토가 될 수 있는 기회를 얻게 되었습니다.

**이 여정이 드러내는 요구사항:**
- 신규 사용자 온보딩 가이드
- 신규 사용자 전용 파티 시스템
- 멘토링 시스템
- 인기도 시스템 설명 및 교육
- 일반 파티 전환 프로세스

### Journey 4: 플랫폼 관리자/운영자 (김영희) - 커뮤니티 품질 관리하기

김영희는 Party 플랫폼의 운영자로, 커뮤니티 품질 관리 및 모니터링, 평가 시스템의 공정성 유지, 악의적 평가 탐지 및 제재, 커뮤니티 가이드라인 관리를 담당합니다.

평일 오전, 김영희는 관리자 대시보드에 로그인하여 커뮤니티 품질 지표를 확인합니다. 신고된 평가를 검토하고, 악의적 평가 패턴을 탐지하는 시스템을 모니터링합니다. 의심스러운 평가 패턴이 발견되면, 해당 평가를 검토하고 필요시 제재 조치를 취합니다.

평가 시스템의 공정성을 유지하기 위해, 김영희는 인기도 회복 메커니즘을 모니터링하고, 사용자 신고를 처리하며, 커뮤니티 가이드라인 위반 사례를 검토합니다. 주간 리포트를 작성하여 커뮤니티 품질 트렌드를 분석하고, 필요시 가이드라인을 업데이트합니다.

**이 여정이 드러내는 요구사항:**
- 관리자 대시보드
- 커뮤니티 품질 모니터링 시스템
- 평가 시스템 공정성 관리 도구
- 악의적 평가 탐지 및 제재 시스템
- 신고 처리 시스템
- 커뮤니티 가이드라인 관리
- 분석 및 리포트 기능

### Journey 5: 멘토 사용자 (박지훈) - 신규 사용자 멘토링하기

박지훈은 30세 경험 많은 파티 참여자로, 인기도가 높고 다양한 파티에 참여한 경험이 많습니다. 그는 신규 사용자 멘토링에 참여하여 커뮤니티 문화를 전달하고 신규 사용자의 성공적인 온보딩을 지원합니다.

멘토링 시스템에 참여 신청한 박지훈은 신규 사용자 전용 파티에 멘토로 배정됩니다. 파티 전, 그는 신규 사용자들에게 파티 문화와 매너에 대해 안내하고, 첫 파티 참여에 대한 긴장을 완화시켜줍니다.

파티 중, 박지훈은 신규 사용자들이 편안하게 참여할 수 있도록 도와주고, 파티 후에는 평가 방법과 인기도 시스템에 대해 설명합니다. 신규 사용자들이 긍정적 평가를 받고 인기도가 상승하는 것을 도와주면서, 그는 커뮤니티 문화의 지속적 전달에 기여합니다.

**이 여정이 드러내는 요구사항:**
- 멘토링 시스템 등록 및 관리
- 멘토-멘티 매칭 시스템
- 신규 사용자 전용 파티 멘토 배정
- 멘토링 가이드 및 자료
- 멘토 활동 추적 및 인센티브 시스템

## Web Application Requirements

### Architecture Approach

**Framework:** Next.js with Server-Side Rendering (SSR)
**Application Type:** Multi-Page Application (MPA)

Party는 Next.js SSR을 사용하여 서버 사이드 렌더링을 구현합니다. 이는 SEO 최적화와 초기 로딩 성능 향상에 유리하며, 파티 목록과 상세 페이지가 검색 엔진에 잘 노출되도록 합니다.

### Browser Support

**Supported Browsers:**
- Chrome (최신 2개 버전)
- Edge (최신 2개 버전)

**Browser Support Strategy:**
- 최신 브라우저만 지원하여 개발 복잡도 감소
- 최신 웹 표준 기능 활용 가능
- 크로스 브라우저 호환성 테스트 범위 축소

### SEO Strategy

**SEO Requirements:**
- 파티 목록 페이지 검색 엔진 노출
- 파티 상세 페이지 검색 엔진 노출
- 메타 태그 및 Open Graph 태그 최적화
- 구조화된 데이터 (Schema.org) 마크업
- 사이트맵 생성 및 제출

**SEO Implementation:**
- Next.js의 SSR 기능을 활용한 서버 사이드 렌더링
- 동적 메타 태그 생성 (파티 제목, 설명 등)
- 검색 엔진 크롤러를 위한 최적화된 HTML 제공

### Real-Time Features

**Real-Time Requirements:** None

Party는 실시간 기능이 필요하지 않습니다. 알림, 채팅, 실시간 업데이트 등의 실시간 통신 기능은 MVP 범위에 포함되지 않으며, 향후 버전에서 고려할 수 있습니다.

### Accessibility

**Accessibility Priority:** Low

접근성은 현재 우선순위가 낮으며, MVP 이후 버전에서 고려할 예정입니다.

### Performance Targets

**Page Load Performance:**
- 초기 페이지 로딩 시간: 3초 이내
- API 응답 시간: 500ms 이내
- 이미지 최적화 및 지연 로딩

**Server-Side Rendering Performance:**
- SSR 렌더링 시간: 200ms 이내
- 정적 페이지 생성 (Static Site Generation) 활용 가능 영역 식별

### Responsive Design

**Responsive Strategy:**
- Mobile-first 접근 방식
- 주요 브레이크포인트: Mobile (~767px), Tablet (768px~1023px), Desktop (1024px+)
- Tailwind CSS를 활용한 반응형 디자인 구현

## Project Scoping & Phased Development

### MVP Strategy & Philosophy

**MVP Approach:** Problem-Solving MVP
**Resource Requirements:** 중간 규모 팀 (프론트엔드, 백엔드, 디자이너)

Party의 MVP는 핵심 문제 해결에 집중합니다. 파티 주최자와 참여자가 신뢰할 수 있는 참여자를 찾고, 파티에 특화된 기능을 제공하는 것이 핵심입니다. 최소한의 기능으로 사용자에게 실질적인 가치를 제공하고, 빠른 검증 학습을 통해 사용자 피드백을 수집합니다.

**MVP 핵심 가치:**
- 파티 전문성: 파티에만 집중한 특화 기능
- 신뢰 시스템: 인기도 기반 평가 시스템
- 핵심 문제 해결: 신뢰할 수 있는 참여자 확보

### MVP Feature Set (Phase 1)

**Core User Journeys Supported:**
- 파티 주최자: 파티 생성 및 참여자 모집
- 파티 참여자: 파티 검색 및 참여 신청
- 신규 사용자: 첫 파티 참여 (기본 기능)

**Must-Have Capabilities:**

1. **파티 생성/검색/관리 도구**
   - 파티 생성: 파티 정보 입력 및 생성 (제목, 설명, 날짜, 시간, 장소, 인원, 카테고리, 이미지)
   - 파티 검색: 카테고리, 날짜, 지역, 키워드 검색
   - 파티 관리: 참여자 관리 (참여 승인/거절), 파티 정보 수정 및 삭제, 파티 상태 관리

2. **인기도 기반 평가 시스템**
   - 인기도 계산: 참여자들의 평가로 인기도 결정 (평균 평가 점수 기반)
   - 인기도 표시: 사용자 프로필, 파티 참여자 목록에 인기도 표시
   - 인기도 기반 필터링: 파티 주최자가 인기도 기준 설정 가능, 인기도가 낮은 참여자의 파티 참여 제한

3. **파티 후 평가/리뷰**
   - 평가 작성: 파티 후 참여자 간 상호 평가 작성 (매너, 참여도, 분위기 기여 등)
   - 리뷰 기능: 파티 및 파티 주최자에 대한 리뷰 작성 및 조회
   - 평가 관리: 평가 이력 조회, 받은 평가 및 작성한 평가 목록, 평가 통계 및 요약

**MVP에서 제외된 기능:**
- 신규 사용자 온보딩 시스템 (신규 사용자 전용 파티, 멘토링 시스템)
- 공정한 평가 시스템 고도화 (악의적 평가 탐지, 인기도 회복 메커니즘)
- 파티 주최자 지원 도구 (참여자 모집 알림, 평가 관리 대시보드)
- 참여 이력 관리 시스템

### Post-MVP Features

**Phase 2 (Post-MVP - Growth):**

1. **신규 사용자 온보딩**
   - 신규 사용자 전용 파티 자동 생성 및 추천
   - 경험 있는 사용자 멘토링 시스템
   - 신규 사용자 가이드 및 튜토리얼

2. **공정한 평가 시스템 고도화**
   - 악의적 평가 탐지 알고리즘 및 자동 제재
   - 인기도 회복 메커니즘 (시간 경과, 추가 긍정 평가)
   - 평가 기준 가이드라인 및 예시 제공
   - 평가 이력 투명성 강화

3. **파티 주최자 지원 도구**
   - 참여자 모집 알림 및 자동화 기능
   - 평가 관리 대시보드
   - 인기도 기준 설정 권한 고도화
   - 파티 성과 분석 도구

**Phase 3 (Expansion):**

1. **참여 이력 관리 시스템**
   - 참여자의 과거 파티 참여 이력 상세 관리
   - 참여 이력 기반 추천 시스템
   - 참여 패턴 분석 및 인사이트 제공

2. **지역 확장**
   - 서울 중심으로 시작하여 전국으로 확장
   - 지역별 커뮤니티 매니저 운영
   - 지역 특성에 맞는 파티 카테고리 제공

3. **플랫폼 확장**
   - 모바일 앱 출시 (iOS 및 Android)
   - 파트너십 및 이벤트 후원
   - 프리미엄 기능 및 수익 모델

4. **고급 기능**
   - AI 기반 파티 추천 시스템
   - 소셜 기능 강화 (친구 시스템, 그룹 기능)
   - 실시간 기능 (실시간 알림, 파티 중 채팅)

### Risk Mitigation Strategy

**Technical Risks:**

**위험:** 인기도 계산 알고리즘의 복잡성
- **완화 전략:** MVP에서는 단순한 평균 평가 점수 기반 알고리즘 사용, 향후 고도화

**위험:** 대규모 사용자 확장 시 성능 이슈
- **완화 전략:** Next.js SSR을 활용한 성능 최적화, 데이터베이스 인덱싱 최적화

**Market Risks:**

**위험:** 사용자 채택 부족
- **완화 전략:** MVP 성공 기준 달성 모니터링, 사용자 피드백 수집 및 빠른 개선

**위험:** 경쟁 플랫폼과의 차별화 부족
- **완화 전략:** 파티 전문성과 인기도 시스템 강조, 사용자 경험 차별화

**Resource Risks:**

**위험:** 개발 리소스 부족
- **완화 전략:** MVP 범위를 엄격히 유지, 핵심 기능에 집중, Post-MVP 기능은 우선순위화

**위험:** 팀 스킬 부족
- **완화 전략:** Next.js, Tailwind CSS 등 검증된 기술 스택 사용, 학습 곡선 최소화

## Functional Requirements

### 사용자 관리

- FR1: 사용자는 계정을 생성할 수 있다
- FR2: 사용자는 프로필 정보를 입력하고 수정할 수 있다
- FR3: 사용자는 자신의 프로필을 조회할 수 있다
- FR4: 사용자는 다른 사용자의 프로필을 조회할 수 있다
- FR5: 사용자는 자신의 인기도를 조회할 수 있다
- FR6: 시스템은 사용자 타입을 구분할 수 있다 (일반 사용자, 멘토, 관리자)

### 파티 관리

- FR8: 파티 주최자는 파티 정보를 입력할 수 있다 (제목, 설명, 날짜, 시간, 장소, 인원, 카테고리)
- FR9: 파티 주최자는 파티 이미지를 업로드할 수 있다

- FR11: 파티 주최자는 파티를 삭제할 수 있다
- FR12: 파티 주최자는 파티 상태를 관리할 수 있다 (모집 중, 모집 완료, 진행 중, 종료)
- FR13: 파티 주최자는 참여 신청을 승인할 수 있다

- FR15: 파티 주최자는 참여자 목록을 조회할 수 있다

## Non-Functional Requirements

### Performance

**Page Load Performance:**
- 초기 페이지 로딩 시간: 3초 이내
- API 응답 시간: 500ms 이내
- 이미지 최적화 및 지연 로딩

**Server-Side Rendering Performance:**
- SSR 렌더링 시간: 200ms 이내
- 정적 페이지 생성 (Static Site Generation) 활용 가능 영역 식별

**Database Performance:**
- 데이터베이스 쿼리 응답 시간: 200ms 이내
- 인덱싱 최적화를 통한 검색 성능 보장

### Security

**Data Protection:**
- 모든 사용자 데이터는 암호화되어 저장되어야 한다
- 전송 중 데이터는 HTTPS를 통해 암호화되어야 한다
- 민감한 정보(비밀번호, 개인정보)는 적절한 해싱 및 암호화를 사용해야 한다

**Authentication & Authorization:**
- 안전한 인증 메커니즘을 사용해야 한다
- 사용자 권한에 따른 접근 제어가 구현되어야 한다
- 세션 관리가 안전하게 처리되어야 한다

**Web Security Standards:**
- 일반적인 웹 보안 표준을 준수해야 한다 (OWASP Top 10 고려)
- SQL Injection, XSS, CSRF 등의 공격에 대한 보호가 필요하다
- 입력 데이터 검증 및 sanitization이 구현되어야 한다

### Scalability

**User Capacity:**
- 시스템은 동시에 1,000명 이상의 사용자를 지원할 수 있어야 한다
- 10배 사용자 성장(1,000명 → 10,000명)을 지원할 수 있어야 한다
- 성능 저하 없이 사용자 수 증가를 처리할 수 있어야 한다

**Traffic Handling:**
- 예상되는 트래픽 패턴을 처리할 수 있어야 한다
- 피크 시간대의 트래픽 증가를 처리할 수 있어야 한다
- 데이터베이스 및 서버 리소스가 효율적으로 사용되어야 한다

**Growth Planning:**
- 수평 확장(horizontal scaling)이 가능한 아키텍처를 고려해야 한다
- 데이터베이스 확장 전략이 수립되어야 한다
- 캐싱 전략을 통한 성능 최적화가 고려되어야 한다
